# Building the Maisha na Lifeza native app (Android)

The website is also an installable app, two ways:

## 1. PWA (zero build — already live)

Visiting the site on Android Chrome shows the **Install Pawa** button
(`js/fab.js`); iOS Safari users get a Share ▸ Add to Home Screen hint.
Everything that makes this work is in the repo: `manifest.json` (PNG icons,
maskable icons, screenshots, shortcuts), `service-worker.js` (offline
app-shell), `icons/` (generated by `node scripts/make_icons.mjs` — rerun it
after changing `icons/icon-maskable.svg`; the dev server must be running for
the screenshot step).

## 2. Native Android APK (Capacitor)

The repo root is the website, so a staging step copies just the app files
into `www/` (gitignored) before each native build:

```powershell
node scripts/build_app.mjs        # stage site -> www/
npx cap sync android              # copy www/ into the android project
cd android
.\gradlew assembleDebug           # debug APK
# -> android\app\build\outputs\apk\debug\app-debug.apk
```

### One-time machine setup

- **JDK 17+** (already installed: Temurin 21).
- **Android SDK** — no Android Studio needed:
  `powershell -ExecutionPolicy Bypass -File scripts\setup_android_sdk.ps1`
  installs cmdline-tools + platform-tools + android-36 + build-tools 36 to
  `%LOCALAPPDATA%\Android\Sdk` and writes `android/local.properties`.

### Release build (Play Store / sideload)

1. Create a keystore once (keep it OUT of the repo — it's gitignored by
   `android/.gitignore` defaults, but store a backup safely):
   `keytool -genkey -v -keystore maishahub.keystore -alias maishahub -keyalg RSA -keysize 2048 -validity 10000`
2. `cd android; .\gradlew assembleRelease`, then sign with `apksigner`, or
   configure `signingConfigs` in `android/app/build.gradle`.
3. App identity: `com.maishahub.app` (set in `capacitor.config.json`).

### If the Gradle wrapper download times out (flaky network)

The first build downloads `gradle-8.x-all.zip` (~130 MB). On a slow link it can
`SocketTimeoutException`. Fetch it once with a resumable downloader and point
the wrapper at the local file:

```powershell
curl.exe -L --retry 5 -o $env:TEMP\gradle-8.14.3-bin.zip `
  https://services.gradle.org/distributions/gradle-8.14.3-bin.zip
# then edit android\gradle\wrapper\gradle-wrapper.properties:
#   distributionUrl=file\:///C\:/Users/<you>/AppData/Local/Temp/gradle-8.14.3-bin.zip
```

Revert `distributionUrl` to the canonical `https://services.gradle.org/...`
before committing so the repo builds anywhere. The Android SDK installer
(`scripts/setup_android_sdk.ps1`) has the same network sensitivity — if it
stalls, the cmdline-tools zip can be fetched with `curl.exe` the same way.

### Updating the app after site changes

Web code ships inside the APK, so after editing HTML/CSS/JS run
`node scripts/build_app.mjs && npx cap sync android` and rebuild. (The PWA
needs no rebuild — users get updates on next visit via the service worker;
remember to bump `VERSION` in `service-worker.js`.)
